Here's the set of default keybindings available in Wave. It is split into sections.
Some keybindings are always active. Others are only active for certain types of blocks.

Note that these are the MacOS keybindings (they use "Cmd"). For Windows and Linux,
replace "Cmd" with "Alt" (note that "Ctrl" is "Ctrl" on both Mac, Windows, and Linux).

Chords are shown with a + between the keys. You have 2 seconds to hit the 2nd chord key after typing the first key. Hitting Escape after an initial chord key will always be a no-op.

## Custom Keybindings

You can remap or disable any built-in action by creating a `keybindings.json` file in your Wave config directory (`~/.config/waveterm/` on Linux/macOS, `%APPDATA%\waveterm\` on Windows).

Changes to `keybindings.json` take effect immediately — no restart required.

### Format

```json
{
  "app:closeblock": { "key": "Ctrl:Shift:q" },
  "app:newtab": { "key": "Cmd:Shift:t" },
  "app:magnify": { "disabled": true }
}
```

Each key in the object is an **action name** (see table below). The value is an object with one or more of:

- `"key"`: a new key descriptor string (replaces the default binding)
- `"disabled": true`: removes the binding entirely
- `"blockdef"`: opens a new block when pressed (see [Block Keybindings](#block-keybindings) below)

### Key Descriptor Format

Key descriptors use colon-separated modifiers followed by the key name, e.g.:

- `"Cmd:t"` — Cmd/Alt + t
- `"Ctrl:Shift:q"` — Ctrl + Shift + q
- `"F2"` — F2

On macOS `Cmd` means the Command key; on Windows/Linux `Cmd` maps to Alt.

| Action Name                               | Default Key              | Description                                 |
| ----------------------------------------- | ------------------------ | ------------------------------------------- |
| `app:newtab`                              | `Cmd:t`                  | Open a new tab                              |

### Block Keybindings

You can bind a key to open a new block directly, without using a named action. Set `"blockdef"` to a block definition object — the same format used by [custom widgets](./customwidgets).

The action name is just a user-defined label (it can be anything) and does not need to match a built-in action.

```json
{
  "my:browser": {
    "key": "Ctrl:Shift:b",
    "blockdef": { "meta": { "view": "web", "url": "https://google.com" } }
  },
  "my:notes": {
    "key": "Ctrl:Shift:n",
    "blockdef": { "meta": { "view": "preview", "file": "~/notes.md" } }
  },
  "my:terminal": {
    "key": "Ctrl:Shift:e",
    "blockdef": { "meta": { "view": "term" } }
  }
}
```

If a `blockdef` entry uses the same key as a built-in action, the `blockdef` takes precedence.
